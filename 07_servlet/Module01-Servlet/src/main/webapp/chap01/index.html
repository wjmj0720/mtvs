서블릿 생명주기

서블릿(Servlet)은 웹 서버에서 실행되는 자바 프로그램으로, 클라이언트 요청을 처리하고 동적인 웹 콘텐츠를 생성합니다. 서블릿의 생명주기는 아래와 같은 단계로 이루어집니다.

1. 초기화 (init)

  • 서블릿 컨테이너가 서블릿 객체를 생성한 후 호출되는 단계입니다.
  • init() 메서드가 실행되며, 서블릿이 필요한 초기 설정을 수행합니다.
  • 이 단계는 서블릿의 생애 동안 단 한 번만 실행됩니다.

2. 서비스 (service)

  • 클라이언트의 요청이 있을 때마다 호출되는 단계입니다.
  • service() 메서드가 요청을 처리하고, HTTP 메서드에 따라 doGet() 또는 doPost() 같은 메서드로 작업을 위임합니다.
  • 이 단계는 요청이 있을 때마다 반복적으로 실행됩니다.

3. 소멸 (destroy)

  • 서블릿이 더 이상 필요하지 않거나 서버가 종료될 때 호출되는 단계입니다.
  • destroy() 메서드가 실행되며, 자원을 정리하거나 마무리 작업을 수행합니다.
  • 이 단계 역시 서블릿의 생애 동안 단 한 번만 실행됩니다.

참고: 서블릿의 생명주기는 서블릿 컨테이너에 의해 관리되며, 개발자는 주로 init(), service(), destroy() 메서드를 오버라이드하여 필요한 로직을 구현합니다.
